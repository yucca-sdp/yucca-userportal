With dataset groups is possible to create logical groupings of datasets

There are two macro types of groups: user-defined groups and special groups that will be used for data operations

User-defined groups

These are groups for total use by the tenant, they are not versioned, and it is possible at any time to add or remove datasets

Special Groups

Special groups are used to determine which datasets must be taken into account for massive data operations
For special groups are versioned, the group is created in draft status, and you can add or remove datasets. When an operation is demand, the group must be consolidated, and from that moment on it will no longer be possible to change. It will however be possible to create a new version

Types of special groups

Pr
Promotion - Promotion is the operation by which data are copied from the datalake to the speed layer (Promotion 01)
Pu
Publication - The publication is the operation with which the data are published on the store (Promotion 02)
Ou
Outgestion - Outgestion is the operation by which data is exported from the datalake
In
Ingestion - Ingestion is the operation by which data is imported into the datalake

Steps to follow to request an operation

Create a group of the desired operation type
Choose the datasets to be processed and add them to the group
Consolidate the group
Contact the Yucca team and agree on how to perform the operation by indicating the group of datasets to consider
In case it was necessary to create a new version of the group and start again

For further details and documentation of the workflow refer to the dedicated section on Knoledge Base Tecnologica
